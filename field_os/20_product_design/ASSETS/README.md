# ASSETS

## What lives here

**Shared design-support material** used across the phase: references to the existing design-system export, exported wireframe/prototype images, screenshots, icons drawn from the system, and other supporting files. This folder stores and references design material; it does not define the design language.

- **Purpose.** Hold shared, reusable material that supports the folder's artifacts.
- **Inputs.** The existing design system (`design-system-documentation.md`, `_ds/`); exports from wireframes, prototypes, and tests.
- **Outputs.** Organized, referenceable assets for use by the other subfolders.
- **Dependencies.** `DESIGN_SYSTEM_INTEGRATION.md` (governs what may be reused/created).
- **Responsible role.** Product Designer.
- **Relationship with the approved architecture.** Support-only. Assets carry no architectural authority and introduce no design language; they reuse the existing system. New visual elements are permitted only under the new-component rule in `DESIGN_SYSTEM_INTEGRATION.md`.
